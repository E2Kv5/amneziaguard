# AmneziaGuard — техническое задание на разработку Android VPN-приложения

> Скопируйте этот документ целиком в Claude Code (или другой AI-инструмент для разработки) как стартовый промт. Перед началом работы уточните package name, название приложения, дизайн-палитру и минимальную поддерживаемую версию Android, если они отличаются от указанных ниже.

## 1. Общая концепция

Разработать нативное Android-приложение — VPN-клиент на протоколе **AmneziaWG 2.0**, со встроенным пофайловым (per-app) фаерволом продвинутого уровня (3 режима + kill-switch + защита от DNS-утечек) и интеграцией в панель быстрых настроек (Quick Settings) Samsung One UI / стокового Android — по образцу официального приложения Amnezia VPN.

## 2. Стек технологий

- **Язык:** Kotlin
- **UI:** Jetpack Compose + Material 3
- **Min SDK:** 26 (Android 8.0), **Target SDK:** актуальный на момент разработки
- **Ядро туннеля:** `android.net.VpnService`
- **Протокол:** [amneziawg-android](https://github.com/amnezia-vpn/amneziawg-android) — официальная userspace-реализация (`amneziawg-go`, встроена через `gomobile bind` в `.aar`), запуск без root
- **QS-тайл:** `android.service.quicksettings.TileService`
- **Хранилище:** Room (конфиги, правила фаервола, профили) + EncryptedSharedPreferences / Android Keystore (ключи)
- **Фоновые задачи:** WorkManager (health-check туннеля, DNS-leak self-test)
- **DI:** Hilt

## 3. Архитектура модулей

```
:app                — точка входа, навигация
:core-tunnel         — обёртка над amneziawg-go, парсинг конфигов, хендшейк, roaming
:core-firewall       — логика per-app routing, kill-switch, DNS-leak protection
:core-data           — Room, DataStore, репозитории
:feature-connect     — главный экран подключения
:feature-firewall    — экран управления приложениями/группами
:feature-settings    — настройки протокола AWG, импорт конфигов
:tile                — QuickSettingsTileService
:background          — Foreground VpnService + уведомление
```

## 4. Интеграция AmneziaWG 2.0

- Парсинг `.conf`-файлов и QR-кодов в формате AmneziaWG, включая параметры версии 2.0: `Jc`, `Jmin`, `Jmax`, `S1`, `S2`, `H1–H4`, `I1–I5` (сигнатурные пакеты, маскирующие handshake под QUIC/DNS/SIP).
- Криптографическое ядро (Curve25519, ChaCha20-Poly1305, Noise IK) не трогаем — оно идентично WireGuard, меняется только транспортный слой (динамические диапазоны заголовков, паддинг, сигнатурные пакеты).
- Реконнект при смене сети (Wi-Fi ↔ LTE) без разрыва пользовательских TCP-сессий.
- Поддержка нескольких сохранённых серверов/конфигов с быстрым переключением.
- Импорт: вручную (вставка текста конфига), из файла, сканированием QR.

## 5. Фаервол — продвинутый уровень + kill-switch + DNS-leak protection

Для каждого установленного приложения — **3 режима**:

1. **Через VPN** — трафик идёт в туннель (`Builder.addAllowedApplication`)
2. **Напрямую** — в обход VPN (`Builder.addDisallowedApplication`)
3. **Без интернета** — приложению запрещён любой выход в сеть

### Важное техническое ограничение и два варианта реализации режима «без интернета»

Штатный `VpnService` умеет управлять только тем, что *входит в сам VPN-интерфейс* (allow/disallow относительно туннеля), но не даёт готового API «полностью отрезать приложение от сети» без дополнительных мер. Заложите в архитектуру оба варианта и дайте пользователю выбор в настройках:

- **Без root (по умолчанию):** приложения из категории «без интернета» принудительно заворачиваются в VPN-интерфейс (`addAllowedApplication`), но на уровне обработки пакетов в туннельном сервисе их трафик определяется (сопоставление соединений по uid через `ConnectivityManager`/сокет-таблицы) и просто не пересылается на сервер — пакеты дропаются локально. Менее элегантно, но работает без повышенных прав.
- **С root / Shizuku (опционально, продвинутый режим):** нативные `iptables`/`nftables` правила с `owner`-match по UID — надёжная и производительная блокировка на уровне ядра. Требует детекции root/Shizuku и деградации до первого варианта при их отсутствии.

Дополнительно:
- **Kill-switch:** при обрыве туннеля — немедленная блокировка исходящего трафика всех приложений из категории «через VPN» (жёсткий VPN-роутинг без реального аплинка), до восстановления соединения.
- **DNS-leak protection:** перехват всех DNS-запросов (UDP:53, DoT:853, DoH) и принудительный редирект через DNS VPN-сервера; периодический self-test (сверка резолвящего IP с ожидаемым) с уведомлением пользователя при обнаружении утечки.
- **Группы/профили приложений:** пресеты вида «Мессенджеры — через VPN», «Банковские приложения — напрямую», «Рекламные SDK/трекеры — блок».

## 6. Quick Settings Tile (интеграция в шторку Samsung)

- `TileService` с иконкой в стиле референса (круглая плашка с логотипом), подпись «AmneziaGuard».
- Тап — toggle подключения (connect/disconnect).
- Долгое нажатие — открытие приложения на экране быстрого выбора сервера.
- Состояния: `Active` (подключено), `Inactive` (отключено), `Unavailable` (ошибка/нет конфига) — с соответствующей иконкой/цветом.
- Обновление состояния в реальном времени через `updateTile()` / `requestListeningState`.
- Регистрация в `AndroidManifest.xml`: `intent-filter` на `android.service.quicksettings.action.QS_TILE` + permission `android.permission.BIND_QUICK_SETTINGS_TILE`.
- Это стандартный публичный Android API — специального Samsung SDK не требуется, тайл автоматически появляется в панели редактирования быстрых настроек One UI, как на референсном скриншоте.

## 7. Экраны (UI/UX)

- **Главный:** крупная кнопка Connect/Disconnect, текущий статус, выбранный сервер, скорость/трафик.
- **Фаервол:** список установленных приложений (иконка + имя) с 3-позиционным переключателем на каждое, поиск, группы/профили.
- **Протокол:** редактор параметров AWG 2.0 (Jc/Jmin/Jmax, S1/S2, H1–H4, I1–I5), импорт конфига (файл/QR/ссылка), список сохранённых серверов.
- **Безопасность:** переключатели kill-switch и DNS-leak protection, кнопка «Проверить на утечку».

## 8. Разрешения

- `BIND_VPN_SERVICE`
- `FOREGROUND_SERVICE`, `FOREGROUND_SERVICE_SPECIAL_USE`
- `BIND_QUICK_SETTINGS_TILE`
- `QUERY_ALL_PACKAGES` (для списка приложений в фаерволе — потребует обоснования в Google Play Data Safety при публикации)
- Опционально: root/Shizuku-доступ для продвинутого режима фаервола

## 9. Безопасность и приватность

- No-logs: история подключений не сохраняется
- Приватные и preshared-ключи — только в Android Keystore / EncryptedSharedPreferences
- Локальное хранилище конфигов — зашифровано

## 10. Этапы разработки

1. MVP: базовый туннель на amneziawg-android, connect/disconnect
2. Простой per-app allow/disallow фаервол (2 режима)

4. Продвинутый фаервол (3 режима), выбор без-root/root реализации
5. Kill-switch и DNS-leak protection

- Ручное тестирование kill-switch (принудительный обрыв туннеля) и DNS-leak self-test
